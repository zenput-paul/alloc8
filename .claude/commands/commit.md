Commit and push changes:

1. Run `/check` first — do not proceed if any checks fail
2. Run `git status` and `git diff` to review all changes
3. Stage relevant files (avoid staging secrets or generated files)
4. Write a descriptive commit message:
   - Summarize the nature of the changes (new feature, bug fix, refactor, etc.)
   - Focus on the "why" rather than the "what"
   - End with: Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>
5. Commit and push to remote
6. Report the commit hash and a summary of what was pushed
